01

Plugin, Skill, Agente: la distinción que nadie explica bien

Los tres términos aparecen juntos constantemente y casi siempre sin definir. El resultado: mucha gente que cree tener un agente y tiene una skill, o que instala plugins pensando que está añadiendo inteligencia a su sistema.

Qué es
Para qué sirve
Ejemplo
Plugin
Añade una capacidad al entorno donde corre el agente — a Claude Code como herramienta, no al agente en sí. Extiende lo que puedes hacer desde el IDE.
Skill Creator — crea y gestiona skills sin salir de Claude Code
Skill
Instrucciones especializadas que el agente carga cuando las necesita. Le dice CÓMO hacer una tarea concreta. Una skill = una tarea. Pasiva por defecto.
copywriting/skill.md — instrucciones de voz, estructura y tono para escribir
Agente
Sistema con un objetivo claro, un modelo asignado y skills disponibles. Se ejecuta solo, itera hasta que el resultado cumple el criterio — sin que tú pulses siguiente en cada paso.
Agente guionista — recibe el briefing, genera el guion completo, lo deja en output/
La relación entre los tres

Los plugins amplían Claude Code como entorno → las skills dan instrucciones especializadas al agente → el agente usa esas skills para completar su objetivo. Cada pieza tiene su función. Mezclarlas no suma capacidad — genera confusión.

02

El cerebro y la mesa de trabajo

Claude Code es el cerebro. El entorno donde lo ejecutas — terminal, VS Code, Antigravity — es la mesa de trabajo. El cerebro no cambia según la mesa. Lo que cambia es el flujo de trabajo que puedes construir alrededor.

Terminal
Línea de comandos nativa
El acceso directo. Sin intermediarios. Ideal para ejecutar agentes, gestionar skills y lanzar proyectos desde cero. Requiere más comodidad con la terminal, pero da el control total.
VS Code
Extensión para el editor de código
Claude Code integrado directamente en VS Code. El agente ve el código que tienes abierto, puede leer el proyecto completo y actúa dentro del contexto del editor. Es el entorno que más facilita la supervisión.
Antigravity
Entorno no-code de Google
La herramienta del Nivel 1 también acepta Claude Code como cerebro. Compatible con el sistema de skills: copiar la carpeta .claude/skills/ a .agents/skills/ y las skills funcionan en Antigravity sin modificar.
Lo que esto cambia para ti

Si ya construiste algo con Antigravity en el Nivel 1, no empiezas desde cero. El cerebro (Claude Code + skills) se puede conectar al entorno que ya usas. La curva de aprendizaje no es de herramienta — es de arquitectura.

03

MCPs: los brazos del agente

Sin MCPs, el agente solo razona. Puede escribir, analizar, estructurar — todo dentro de su ventana de contexto. Con MCPs, actúa: guarda archivos, sincroniza repositorios, genera imágenes, recupera información de tu nube de conocimiento.

Un MCP (Model Context Protocol) es un protocolo estándar que permite a Claude conectarse con herramientas externas. No es magia — es una interfaz definida que el agente puede llamar como si fuera una skill, pero que vive fuera del modelo.

Red de conocimiento
Google Drive MCP
El agente lee y escribe en Drive directamente. Archivos de contexto, referencias, outputs — todo en la misma nube que ya usas. Sin copiar y pegar manualmente entre el agente y tu almacenamiento.
conocimiento
NotebookLM MCP
NotebookLM convierte documentos en bases de conocimiento consultables. El MCP permite al agente recuperar información de esa base durante la ejecución — sin necesidad de pasarle el documento completo como contexto.
conocimiento
Obsidian MCP
Obsidian es un sistema de notas con grafo de conexiones entre ideas. Combinado con Drive y NotebookLM, forma una red neuronal de conocimiento personal: el agente puede depositar outputs y recuperar contexto de esa red en la misma sesión.
conocimiento

Drive + NotebookLM + Obsidian trabajan juntos. El agente accede a la red completa, no a cada herramienta por separado.

Acción directa
GitHub MCP
El agente hace commits, abre pull requests y sincroniza cambios directamente en el repositorio. Sin salir de Claude Code, sin copiar código manualmente. Cada cambio que el agente produce queda versionado en el repo automáticamente.
acción
Traducción IA a IA
Higgsfield MCP
Higgsfield es una plataforma de generación de video por IA. El MCP permite que Claude traduzca tu intención en el prompt técnico óptimo para Higgsfield — y que lo envíe directamente. El resultado es más preciso que si escribieras el prompt tú mismo, porque dos IAs se comunican en el mismo idioma técnico.
IA a IA
La traducción más precisa no ocurre entre humano y máquina.
Ocurre cuando una IA le habla a otra.
04

Los archivos de contexto: la capa que define quién eres

El CLAUDE.md define las instrucciones generales del proyecto. Las skills definen cómo hacer tareas concretas. Los archivos de contexto en context/ responden a una pregunta distinta: ¿quién eres, a quién le hablas y con qué referencias trabaja el agente?

No son obligatorios para que el agente funcione. Son obligatorios para que el agente funcione bien.

📁 tu-proyecto/
├── CLAUDE.md ← instrucciones generales del proyecto
├── context/ ← quién eres + a quién le hablas
│  ├── perfil.md ← tu voz, tu tono, cómo trabajas
│  ├── audiencia.md ← a quién va dirigido el output
│  └── referencias/ ← ejemplos de lo que quieres
├── output/ ← donde el agente deja resultados
└── .claude/skills/ ← instrucciones especializadas
CLAUDE.md vs context/ — no es lo mismo

El CLAUDE.md le dice al agente qué tiene que hacer y cómo operar en este proyecto. Los archivos de context/ le dicen con qué materiales trabaja — tu voz, tu audiencia, tus referencias visuales. Sin context/, el agente produce algo técnicamente correcto pero sin personalidad propia.

05

Permisos: lo que autorizas al agente a hacer

Cuando lanzas un agente por primera vez, Claude Code te pregunta qué puede hacer. La mayoría acepta todo sin leer. Eso no es un problema si sabes qué estás aceptando — pero si no lo sabes, puedes terminar con un agente que modifica archivos que no debería o que gasta créditos en MCPs que no necesitaba.

SÍ
Leer archivos del proyecto
El agente necesita leer CLAUDE.md, context/ y cualquier referencia para hacer su trabajo. Sin este permiso, trabaja a ciegas.
SÍ
Escribir en output/
El agente necesita dejar sus resultados en algún sitio. La carpeta output/ es el lugar diseñado para eso — y el lugar desde donde otros agentes recogen el contexto.
↓
Ejecutar comandos de terminal
Solo si el agente necesita scripts o automatizaciones. Para la mayoría de agentes de contenido o análisis, no es necesario. Actívalo cuando sepas exactamente por qué lo necesitas.
↓
Acceder a la red (MCPs externos)
Necesario si usas MCPs como GitHub, Drive o Higgsfield. Sin este permiso los MCPs no pueden conectarse. Actívalo cuando tengas MCPs configurados — no antes.
NO
Modificar archivos fuera del proyecto
Sin una razón muy concreta, no. Un agente que puede escribir fuera de su carpeta de trabajo puede causar cambios donde no los esperas. El scope del agente debe coincidir con el scope del proyecto.
