El sistema en una frase

Un conjunto de skills coordinadas que saben qué hacer, cuándo hacerlo y cómo encajar entre ellas. No herramientas sueltas — un equipo que funciona cada vez que lo necesitas.

02

El circuito de producción

Cada recurso de esta biblioteca sigue el mismo proceso. Sin excepciones. Lo que cambia es el contenido que entra — el circuito siempre es el mismo.

1
Brief del recurso
Se define el ángulo, el tag (mito/concepto/herramienta/acción/caso real), el arco narrativo y lo que tiene que quedar claro al lector. Sin brief, no hay recurso.
Humano
2
Skill activa según la tarea
Claude Code lee el CLAUDE.md del proyecto y activa la skill correcta según el trabajo: diseño, contenido, estrategia, contexto. Cada skill sabe su rol exacto sin que tengas que decírselo en cada sesión.
Sistema
3
Build del recurso HTML
Se genera el recurso completo: cover SVG, secciones, componentes visuales, checklist, navegación. El sistema mantiene coherencia visual con todos los recursos anteriores de forma automática.
Sistema
4
Doble revisión
Dos checks siempre: veracidad (todo lo que se afirma debe ser verificable) y SEO (meta description, keywords, legibilidad). Si algo no pasa el check, se corrige antes de continuar.
Humano
5
Deploy y enlace
El recurso se añade al catálogo del índice y se actualizan los enlaces de navegación. La biblioteca crece de forma ordenada, sin que tengas que rastrear manualmente qué apunta a qué.
Sistema
Lo que el humano hace — y no hace

El brief, la revisión de veracidad y las decisiones de ángulo son humanas. El diseño, el CSS, los SVG, la redacción, el SEO, la coherencia global — lo gestiona el sistema. El sistema no elimina el criterio. Lo amplifica.

03

El kit — 5 skills, 2 orígenes

Estas son las skills activas en este proyecto ahora mismo. Cuatro vienen de fuera — del marketplace o de GitHub. Una la construí yo. Ese equilibrio es el modelo híbrido: descargas lo que ya existe bien hecho, construyes lo que necesitas y no encuentras.

Descargadas — marketplace y GitHub
U
ui-ux-pro-max GitHub · local
Gestiona el diseño de cada recurso: paleta de colores, tipografía, componentes CSS, SVG, estructura visual. Instalada localmente en el proyecto para que sus reglas apliquen solo aquí.
Sin ella: diseñador o días de prueba-error para que 15 páginas sean coherentes.
C
copywriting Marketplace · global
Estructura el contenido de cada recurso para que sea útil, directo y fácil de leer. Redacta con el tono de alex.decode: anti-hype, sin relleno, números reales.
Sin ella: copywriter o 5+ borradores para que el texto no suene a IA genérica.
S
content-strategy Marketplace · global
Mantiene la coherencia del arco entre niveles. Se asegura de que cada recurso encaje en su nivel y que el conjunto tenga sentido de principio a fin.
Sin ella: estratega de contenido o una hoja de cálculo siempre desactualizada.
P
prompt-master Marketplace · global
Construye y mantiene el contexto del agente. En este proyecto ese contexto vive en el CLAUDE.md — el archivo que le dice al sistema quién es alex.decode, qué construye y cómo opera.
Sin él: brief completo repetido en cada sesión, o un agente sin memoria que no sabe quién eres.
Construida por mí — skill personalizada
★
code-context-reviewer Construida · Skill Creator
Analiza el contexto acumulado en la sesión y limpia lo que ya no aporta información nueva. Un proyecto con 15 recursos, memoria persistente y decisiones acumuladas puede contaminar las respuestas del agente si el contexto no se gestiona. Esta skill lo revisa y lo ajusta para que el sistema siga operando con precisión.
Sin ella: respuestas inconsistentes conforme avanza el proyecto, o sesiones que empiezan desde cero porque el contexto es demasiado pesado.
Por qué una skill construida y no descargada

No existe una skill en el marketplace que gestione específicamente el contexto de este proyecto con su estructura de memoria, su CLAUDE.md y sus 15 recursos. Las skills genéricas de gestión de contexto no saben lo que sabe esta. Eso es exactamente lo que viste en 2.4: cuando no existe lo que necesitas, lo construyes.

Lo que no es una skill — y es lo más importante
CLAUDE.md + memoria del proyecto
Las skills saben cómo hacer las cosas. El CLAUDE.md les dice qué construyen y para quién. Sin él, cada skill opera en genérico. Con él, saben que trabajan para alex.decode, que el tono es anti-hype, que el ratio es 60/40, que el stripe superior es morado para casos reales.

La memoria del proyecto añade continuidad: decisiones tomadas, estado de cada recurso, componentes CSS existentes. El sistema recuerda lo que se ha construido sin que tengas que repetirlo cada vez.

Los números reales

15 recursos · ~2h cada uno · $77/mes de stack. Sin el sistema, estimación conservadora: 1-2 días por recurso si sabes lo que haces, semanas si no. El sistema no es un atajo — es hacer más con el mismo criterio.

Lo que no significa

El sistema no elimina el pensamiento humano — lo amplifica. Alguien tiene que definir el brief, decidir el ángulo y verificar la veracidad. Lo que el sistema elimina es el tiempo de ejecución. Esa es la diferencia entre automatizar trabajo y delegar el criterio.

05

Cómo replicarlo — sin copiar el mío

Un sistema de skills no se copia — se construye para lo que tú produces. El de alex.decode funciona para una biblioteca de recursos HTML. El tuyo tiene que encajar con tu flujo real.

El proceso es el mismo que has visto en este nivel: identifica las tareas que repites, busca si existe una skill en el marketplace para cada una, instala las que encajan, construye las que no existen con Skill Creator, y define tu CLAUDE.md como cerebro del proyecto.

La pregunta que tienes que responder

¿Qué haces tú que se repite semana a semana y podría tener una skill asignada? Esa es la primera skill de tu kit. Una sola. Empieza ahí, no con cinco a la vez.
