01

La mayoría trabaja con IA como si fuera un buscador

Abres Claude Code, describes lo que necesitas, obtienes algo útil. Cierras. Al día siguiente vuelves a abrir y el modelo no recuerda nada del proyecto, del contexto ni de cómo te gusta trabajar. Cada sesión desde cero — como si fuera la primera vez.

No es un fallo del modelo. Es que nadie le dio el contexto para trabajar de otra forma.

El problema de fondo

Pedirle a Claude Code que construya algo sin contexto es como pedirle a alguien que lleva dos horas en la empresa que entregue el proyecto de la tarde. Sin briefing, sin instrucciones, sin historial de lo que ya existe. El resultado va a ser genérico — no porque el modelo sea malo, sino porque no sabe qué construir ni para quién.

02

No es el prompt lo que decide el resultado — es el entorno que construyes antes

Los desarrolladores que construyen con IA de forma consistente no tienen mejores prompts. Tienen mejor contexto. Esto es lo que hay detrás.

/init
Orientación inicial
Claude Code lee la carpeta, procesa el CLAUDE.md y entiende qué tiene delante. Sin este paso, el modelo entra a ciegas en cada sesión.
CLAUDE.md
Instrucciones permanentes
Rol del agente, stack, tono, protocolo de trabajo, skills disponibles. Todo lo que el modelo necesita saber sin que lo repitas en cada sesión.
MEMORY.md
Contexto que evoluciona
Lo que no está en el código pero importa entre sesiones: decisiones tomadas, estado actual del proyecto, preferencias confirmadas.
/compact
Gestión activa del contexto
El rendimiento cae visiblemente a partir de los 100k tokens. Compacta antes de llegar ahí — el contador de tokens está visible en la barra inferior de Claude Code.
Skills / MCPs
Capacidades extendidas
Copywriting, estrategia de contenido, búsqueda web, análisis. No reinventas capacidades — activas las que ya existen cuando las necesitas.
No construyes con IA. Construyes con IA en un entorno que has diseñado.
03

Ejemplo real: así está construida esta biblioteca

Lo que estás leyendo existe porque hay estructura detrás — meses de trabajo en sesiones que no pueden perderse entre sí.

Capa Qué hace en este proyecto
CLAUDE.md Define el rol del agente, el stack visual, las skills activas, el protocolo de compactación y el idioma
MEMORY.md Guarda el estado de cada nivel (completo/pendiente), componentes CSS por recurso, decisiones de diseño y el próximo paso
/compact Se ejecuta aprox. cada 100k tokens — el rendimiento baja notablemente a partir de ahí. El contador de tokens está visible en la barra inferior de Claude Code.
Skills Copywriting para los recursos, grill-me para clarificar briefs, content-strategy para el arco de cada nivel

Para arrancar en cualquier proyecto nuevo, estos son los dos primeros movimientos:

Paso 1 — Orientar a Claude Code en la carpeta del proyecto

Paso 2 — Compactar aprox. cada 100k tokens (contador visible en la barra inferior de Claude Code)

La estructura mínima para empezar

Carpeta del proyecto → ejecuta /init → crea CLAUDE.md con rol, stack y protocolo → empieza a construir. El resto lo vas añadiendo según el proyecto lo pide.

04

4 preguntas para diagnosticar cualquier proyecto

Revisa el proyecto que tienes abierto ahora mismo — o el último en el que trabajaste con IA.

01
¿Tu carpeta de proyecto tiene un CLAUDE.md con instrucciones reales?
Si no existe o está vacío, el modelo no tiene contexto permanente. Cada sesión empieza sin saber quién eres ni qué construyes.
02
¿Ejecutaste /init en esa carpeta antes de abrir el chat?
Sin este paso, Claude Code no procesa el proyecto. Es pedirle que trabaje sin haberse leído los apuntes.
03
¿El contexto activo de tu sesión es útil o hay ruido acumulado?
En la práctica, el rendimiento baja visiblemente a partir de ~100k tokens de contexto. Compacta antes de llegar ahí — no esperes a que el modelo dé respuestas inconsistentes para actuar.
04
¿Estás usando el modelo correcto para la tarea?
Haiku para respuestas rápidas y ediciones menores. Sonnet para estrategia, borradores y análisis. Opus para investigación profunda o sistemas complejos.
Diagnóstico inmediato
Lo que voy a corregir esta semana…

Si respondes "no" a cualquier pregunta, ya tienes el plan.

05

El entorno ya está diseñado. Las herramientas vienen después.

Las 5 capas no son una configuración técnica. Son el contrato de trabajo entre tú y el modelo. Sin ese contrato, cada sesión empieza desde cero. Con él, el modelo sabe quién eres, qué construyes y cómo quieres trabajar — incluso después de meses.
