01

Por qué estos conceptos y no otros

No necesitas saber qué es un frontend, un backend ni una API para empezar a construir con IA. Eso llega solo, cuando lo tocas. Lo que sí necesitas ahora son cuatro palabras que van a aparecer constantemente en todo lo que sigue — y que la mayoría de tutoriales asumen que ya sabes.

Igual que antes de cocinar necesitas saber qué es una sartén y qué es un horno — no para ser chef, sino para no quemarte la primera vez. Este recurso es eso.

La diferencia clave

Un modelo es el cerebro (Claude, GPT, Gemini). Un entorno es la interfaz donde trabajas (Claude Code, Cursor). Un agente es el sistema que usa el cerebro para alcanzar un objetivo. Un skill le enseña al sistema una habilidad nueva. Son cuatro cosas distintas.

02

MCP — el protocolo que conecta la IA con tus herramientas

Hasta hace poco, la IA solo podía ver texto. Le enviabas un mensaje, te respondía. Fin. No podía abrir tu calendario, leer tus archivos o consultar una base de datos en tiempo real.

MCP cambia eso.

Model Context Protocol (MCP)

Estándar abierto creado por Anthropic en noviembre de 2024 y adoptado por OpenAI, Google DeepMind y Microsoft. Define cómo los modelos de IA se conectan con herramientas externas — archivos, calendarios, bases de datos, APIs — de forma segura y estandarizada.

Analogía: MCP es como el puerto USB. Antes de que existiera, cada dispositivo tenía su propio conector. Con MCP, todo se conecta igual, sin importar el fabricante.

Sin MCP, la IA solo puede hablar. Con MCP, puede actuar: leer un documento, escribir en una hoja de cálculo, consultar el tiempo, reservar algo. La diferencia es la misma que entre un asesor que te da consejos y uno que también puede ejecutarlos.

Por qué importa ahora

Las herramientas que vamos a usar en este nivel (Antigravity, Stitch) se conectan con la IA a través de protocolos similares. Cuando oigas "integración" o "conector", estás hablando de la misma idea que MCP.

03

Agente de IA — el loop que no para hasta conseguirlo

Un chatbot responde. Un agente actúa. La diferencia no es el modelo que usan — es lo que hacen con él.

Cuando le das un objetivo a un agente, no espera tu confirmación en cada paso. Planifica, ejecuta, evalúa el resultado y corrige. Si no ha conseguido el objetivo, vuelve a empezar. El loop no para hasta que la iteración es correcta — o hasta que le dices que pare.

Loop iterativo del agente

Esto es importante entenderlo antes de tocar las herramientas del nivel 1. Cuando Antigravity genera una app, no lo hace de un golpe — itera: genera, evalúa, corrige. Tú ves el resultado final. El loop ocurrió entre bastidores.

Lo que cambia para ti

Con un agente no describes lo que quieres paso a paso. Describes el objetivo y dejas que el sistema decida cómo llegar. Tu trabajo es evaluar el resultado y redirigir si no es lo que esperabas.

04

Skills y Kits — ampliar lo que puede hacer la IA

Un modelo de IA sabe muchas cosas de forma genérica. Una skill le enseña a hacer algo específico, con un comportamiento concreto y repetible.

Skill

Extensión modular que añade una capacidad específica a Claude Code. Se almacena en el directorio ~/.claude/skills/ y se activa con un comando. Contiene instrucciones estructuradas en un archivo SKILL.md.

Analogía: como instalar un plugin en un navegador. El navegador ya funciona, el plugin le añade algo que antes no podía hacer.

La pregunta correcta

No es "¿cuál es mejor?" Es "¿qué quiero hacer?" Si quieres construir apps sin tocar código — como hacemos en este nivel con Antigravity — ninguno de los tres es necesario ahora mismo. Son herramientas para cuando empieces a trabajar con código directamente.

FAQ

Preguntas frecuentes

¿Qué es MCP en inteligencia artificial? ▾
Estándar abierto creado por Anthropic (noviembre 2024) y adoptado por OpenAI, Google y Microsoft. Permite que los modelos de IA accedan a herramientas externas — archivos, calendarios, bases de datos — de forma segura. Sin MCP la IA solo habla. Con MCP puede actuar.
¿Qué diferencia hay entre un chatbot y un agente de IA? ▾
Un chatbot responde. Un agente actúa. El agente tiene un objetivo, planifica cómo alcanzarlo, ejecuta acciones y evalúa el resultado. Si no funciona, corrige y repite — sin que tú tengas que intervenir en cada paso.
¿Claude Code, Cursor y GitHub Copilot son lo mismo? ▾
No. Son entornos de trabajo distintos — la interfaz donde interactúas con la IA. Claude Code es un CLI de terminal (solo Claude). Cursor es un editor de código con múltiples modelos. Copilot es un plugin para tu editor actual. El modelo es el cerebro; el entorno es el espacio de trabajo.
¿Necesito entender todos estos conceptos para construir la app del Nivel 1? ▾
No para ejecutar — sí para entender qué está pasando mientras lo haces. Antigravity y Stitch usan agentes internamente. Saber que existe el loop iterativo te ayuda a interpretar por qué el resultado no es instantáneo y cómo redirigirlo cuando no es lo que esperabas.
